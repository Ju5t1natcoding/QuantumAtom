## About

A real-time 3D visualizer of quantum atomic orbitals, built from scratch in C++17 using SFML. Renders electron probability clouds for multiple atoms using the actual hydrogen-like wave functions, including ground state and excited state orbitals.

## Features

- Quantum orbital visualization using rejection sampling on |ψ|²
- 7 atoms: H, He, Li, C, N, O, Ne
- Ground state and excited state orbitals per atom
- Background loading with per-atom progress tracking
- Interactive 3D rotation (click + drag) and zoom (scroll)
- Auto-rotation toggle
- Ground-state-only view (A) and all-orbitals view (B)
- Animated orbital appearance (points materialize gradually)
- Nucleus with glow effect
- Screenshot export (S key)
- Atom selection menu with loading status

## Project structure

```
QuantumAtom/
  main.cpp          — entry point, rendering loop, menu and simulation states
  Atom.hpp          — AtomData and OrbitalInfo structs
  AtomFactory.hpp   — defines all atoms and their orbitals
  Loader.hpp        — background loading via std::thread
  WaveFunction.hpp  — wave function implementations (1s, 2s, 2p, 3d, 4f)
```

## How to compile and run

Requirements: g++ with C++17 support (MinGW/MSYS2), SFML 2.6

Install SFML (MSYS2):
```
pacman -S mingw-w64-ucrt-x86_64-sfml
```

Compile:
```
g++ -std=c++17 main.cpp -o atom -lsfml-graphics -lsfml-window -lsfml-system
```

Run:
```
./atom
```

## How it works internally

**Wave functions:** Each orbital is defined by a wave function ψ(x, y, z) derived from the hydrogen-like solutions to the Schrödinger equation. The visualized quantity is |ψ|² — the probability density of finding the electron at a given point in space. For atoms with Z > 1, the wave functions are scaled by the atomic number Z, which contracts the orbital toward the nucleus.

**Rejection sampling:** Points are generated by sampling random positions in 3D space and accepting or rejecting each one with probability proportional to |ψ|² / max(|ψ|²). This produces a point cloud whose density matches the actual electron probability distribution.

**Rendering:** Each accepted point is projected from 3D to 2D using rotation matrices (applied via mouse drag or auto-rotation). Depth (the z coordinate after rotation) is encoded as brightness — points closer to the viewer appear brighter, giving a sense of depth without a full 3D renderer.

**Background loading:** When an atom is selected from the menu, its orbitals are generated on a separate `std::thread` via `loadAtomAsync()`. The main thread continues rendering the menu and polling for events while loading proceeds. An `std::atomic<int>` counter tracks progress without requiring a mutex, and `std::atomic<bool>` signals when loading is complete.

**Atom menu:** The application uses a simple state machine with two states — `MENU` and `SIMULATION`. The menu polls `atom.ready` and `atom.loadProgress` each frame to display live loading status for each atom.
